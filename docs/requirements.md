## 機能要件

### MVP（まず作るもの）

#### 1. 単語学習

- 単語カードを表示し、4択から意味を選ぶ形式
- 正解/不正解を記録
- MVP後に答え方のバリエーション（自己申告式など）を追加予定

#### 2. 文法問題

- 4択形式の文法問題
- 解答後に正解・解説を表示

#### 3. 復習スケジューリング（間隔反復）

- 間違えた問題・苦手な問題を自動でスケジューリング
- アルゴリズム: **SM-2**（Anki方式）を採用
  - 正解するほど次の出題間隔が伸びる（1日→3日→7日→...）
  - 不正解で間隔リセット
- SM-2 スコアの割り当て：正解=4、不正解=1（単語・文法ともに共通）

#### 4. AI フィードバック

- 「フィードバックを見る」ボタン押下時に Claude API でフィードバックを生成
  - 例：「なぜ間違えたか」「覚え方のヒント」「似た表現の紹介」
- API エラー・タイムアウト時は「再読み込み」または「スキップ」を選択できるフォールバック UI を表示（学習セッションは止めない）

#### 5. 今日のダッシュボード

- 今日の学習ノルマ（復習 + 新規問題）の件数表示
  - MVP では新規 10 問/日 固定
  - MVP後にユーザーが変更できる設定を追加予定
- 学習継続日数（ストリーク）

## 技術スタック

| 領域           | 技術                           | 理由                                     |
| -------------- | ------------------------------ | ---------------------------------------- |
| フロントエンド | Next.js 16 + React 19          | 既存構成                                 |
| スタイリング   | Tailwind CSS v4                | 既存構成                                 |
| DB             | SQLite + Prisma                | 型安全なORM、マイグレーション管理が楽    |
| API            | Next.js API Routes             | サーバー/クライアント一体で管理しやすい  |
| AI             | Claude API（claude-haiku-4-5） | 速度・コスト面でフィードバック用途に最適 |
| 問題データ     | シードデータ（JSON or SQL）    | MVP は単語・文法各 100 問、完成後に増量  |
| ユニットテスト | Vitest                         | React コンポーネント・ロジックのテスト   |
| E2E テスト     | Playwright                     | 学習フロー全体の動作確認                 |

## データモデル（概要）

```
Question（問題）
  - id, type（単語/文法）, content, choices, answer, explanation

UserAnswer（回答履歴）
  - id, questionId, isCorrect, answeredAt

ReviewSchedule（復習スケジュール）
  - id, questionId, nextReviewAt, interval, easeFactor（SM-2パラメータ）
```

- ログイン機能は現時点では不要
- ユーザーIDは固定値（`local-user`）でシンプルに運用
- 将来的にログインを追加する場合は **Google ログイン（Auth.js）** を想定。その際は DB もクラウド（Postgres 等）への移行が必要になる

## 画面構成

```
/               ← 今日のダッシュボード
/study          ← 学習セッション（問題出題 → AI フィードバック）
/review         ← 今日の復習キュー
/progress       ← 学習履歴（Phase 2以降）
```
